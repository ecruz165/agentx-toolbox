Generate `design/foundations/i18n.pen` and write the i18n tokens to
`@theme`. This foundation ensures the design system is **structurally
internationalization-ready** even before specific translations exist —
logical properties replace directional ones, script-aware fonts
declare fallback stacks, and RTL-flip rules apply to directional
icons.

For multilingual K-12 ed-tech products
this isn't optional — the bilingual parent communications, RTL
languages in some districts (Arabic, Hebrew), and the FERPA
requirement to communicate in families' preferred language all
depend on this foundation being correct from day one.

## Pre-flight

1. Read `product/strategy/_context.md` and `product/design/_context.md`.
2. Read `product/.pencil-brand.json` for the chosen `fontDisplay` /
   `fontBody` / `fontMono` (this command extends them with script-
   appropriate fallbacks).
3. Locate the `@theme` source CSS file.
4. Resolve `--scripts`:
   - Default: `latin, latin-ext` (Western European coverage).
   - Common multilingual sets: `latin, latin-ext, cyrillic` (Eastern
     Europe + Latin America), `latin, arabic, hebrew` (RTL coverage),
     `latin, cjk-jp, cjk-kr, cjk-cn` (East Asia), `latin, devanagari,
     bengali, tamil` (South Asia).
   - For K-12 ed-tech specifically: typical district language coverage
     is `latin, latin-ext, arabic, vietnamese, simplified-chinese,
     spanish-extended` — ask which fits your deployment.

## Token set written to `@theme`

```css
@theme {
  /* Script-aware font stacks. Each language family adds appropriate
     fallbacks before the system stack. The browser picks the first
     family that contains the required glyphs. */
  --font-body:
    "{{fontBody}}",                      /* primary brand font */
    "Noto Sans",                         /* universal fallback */
    "Noto Sans Arabic",                  /* arabic + RTL */
    "Noto Sans Hebrew",                  /* hebrew + RTL */
    "Noto Sans CJK JP",                  /* japanese */
    "Noto Sans CJK KR",                  /* korean */
    "Noto Sans CJK SC",                  /* simplified chinese */
    "Noto Sans Devanagari",              /* hindi, marathi */
    "Noto Sans Bengali",                 /* bengali */
    "Noto Sans Tamil",                   /* tamil */
    system-ui, -apple-system, sans-serif;

  /* RTL-aware logical-property tokens — components reference these
     instead of left/right so they auto-flip in RTL layouts.
     CSS logical properties handle the actual flip natively; these
     tokens make the intent visible in component source. */
  --space-inline-start:    var(--space-md);   /* maps to padding-inline-start */
  --space-inline-end:      var(--space-md);   /* maps to padding-inline-end */
  --space-block-start:     var(--space-md);   /* maps to padding-block-start */
  --space-block-end:       var(--space-md);   /* maps to padding-block-end */

  /* RTL-flip transform for directional icons (chevron-right, arrow-back, etc) */
  --i18n-icon-flip-rtl: scaleX(-1);

  /* Locale-aware number formatting — for thousand/decimal separators */
  /* (CSS doesn't format numbers; these tokens flag intent and
     downstream JS uses them for Intl.NumberFormat options) */
  --i18n-number-grouping:  "auto";   /* let Intl.NumberFormat pick per locale */
  --i18n-currency-display: "symbol"; /* "symbol" | "code" | "name" */

  /* Date format token alias (consumed by app code, not CSS) */
  --i18n-date-format-short: "P";       /* date-fns / Intl placeholder */
  --i18n-date-format-long:  "PPP";

  /* Locale-aware quotation marks (rendered by content, not CSS — the
     token is informational and used by content-rendering helpers) */
  --i18n-quote-open:  "“";
  --i18n-quote-close: "”";

  /* Reading direction default (each <html dir="…"> overrides) */
  --i18n-text-direction: ltr;
}

[dir="rtl"] {
  --i18n-text-direction: rtl;
}
```

## Font loading — the half that's not just CSS variables

Declaring fallback chains in `--font-body` is necessary but not
sufficient. The Noto fonts referenced in the chain (`Noto Sans
Arabic`, `Noto Sans CJK JP`, etc.) must actually be loaded for the
browser to use them. This is where most multilingual implementations
silently fail — the CSS chain looks right but the browser falls
through to the system stack and renders boxes / Times New Roman.

This command writes `@font-face` declarations and / or vendor
imports for every script in `--scripts`. Pick **one** strategy per
project; mixing produces inconsistent rendering across devices.

### Strategy 1 — Google Fonts CSS imports (simplest)

Add to `<head>` or top of `@theme` source CSS:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans:wght@400;500;600;700&family=Noto+Sans+Arabic:wght@400;500;600;700&family=Noto+Sans+SC:wght@400;500;600;700&family=Noto+Sans+JP:wght@400;500;600;700&display=swap" rel="stylesheet">
```

The exact `family=` list is generated from `--scripts`. Each script
adds one Noto family to the import. Use `display=swap` so text
remains visible during font load (avoid FOIT).

Pros: easiest to maintain, CDN-cached globally, supports all Noto
scripts. Cons: external dependency (Google), TLS handshake to a
third party — some compliance contexts (EU GDPR-strict, certain
healthcare deployments) prohibit this. Use Strategy 2 in those cases.

### Strategy 2 — Self-hosted Noto fonts

Download the Noto subsets needed and serve from the project's own
CDN / static directory. Add `@font-face` declarations in the
`@theme` source CSS:

```css
/* Generated per --scripts. Example for arabic + simplified-chinese: */
@font-face {
  font-family: "Noto Sans Arabic";
  src: url("/fonts/NotoSansArabic-Variable.woff2") format("woff2-variations");
  font-weight: 100 900;
  font-display: swap;
  unicode-range: U+0600-06FF, U+0750-077F, U+08A0-08FF, U+FB50-FDFF, U+FE70-FEFF;
}

@font-face {
  font-family: "Noto Sans SC";
  src: url("/fonts/NotoSansSC-Variable.woff2") format("woff2-variations");
  font-weight: 100 900;
  font-display: swap;
  unicode-range: U+4E00-9FFF, U+3400-4DBF, U+F900-FAFF;
}
```

The `unicode-range` declaration is critical — it tells the browser
to *only* load the Arabic font when the page actually contains
Arabic characters, *only* load CJK when CJK is present. Without
this, every page downloads every script's font (multiple MB of
unnecessary download).

Pros: no third-party dependency, full control, complies with strict
data-residency requirements. Cons: ~50–500 KB per script in your
build/CDN, requires keeping fonts updated when Noto releases new
versions.

### Strategy 3 — Adobe Fonts kit (if brand is on Adobe)

If the brand uses Adobe Fonts for primary typography (per
`fonts-select`'s vendor strategy), the same kit can include Noto-
equivalent script fonts (Adobe Fonts catalog includes Source Han
Sans for CJK, Mirza for Arabic, etc.). The kit's `<script>` tag
in `<head>` handles all loading.

Substitute the Noto family names in `--font-body` with the Adobe
equivalents (e.g. `Source Han Sans JP` instead of `Noto Sans CJK JP`).

### Loading verification

After fonts are loaded, verify with the FontFace API:

```js
// Run in dev mode to confirm fonts loaded
document.fonts.ready.then(() => {
  const required = ['Inter', 'Noto Sans Arabic', 'Noto Sans SC'];
  const loaded = [...document.fonts].map(f => f.family);
  const missing = required.filter(r => !loaded.includes(r));
  if (missing.length) console.warn('Fonts missing:', missing);
});
```

The audit's i18n plane (when wired) verifies this in CI by checking
that every script in `--scripts` has a corresponding `@font-face`
or import declaration in the `@theme` source CSS.

## RTL flip rules — which icons flip, which don't

A subtle but critical part of i18n design. Some icons must flip when
the document direction is RTL (chevrons, arrows, "back" indicators);
others must not (clock, mute, play). The rule:

- **Directional icons flip**: `chevron-left`, `chevron-right`,
  `arrow-left`, `arrow-right`, `arrow-back`, `arrow-forward`,
  `caret-left`, `caret-right`, `move-left`, `move-right`,
  `redo`, `undo`, `next`, `previous`
- **Functional icons don't flip**: `play`, `pause`, `stop`, `record`,
  `volume-mute`, `volume-low`, `volume-high`, `clock`, `clock-rewind`,
  `search`, `info`, `warning`, `error`, `success`
- **Numerals never flip** even in RTL contexts — Arabic and Hebrew
  numerals read left-to-right inside otherwise-RTL text

Components apply the flip via:

```tsx
<ChevronRightIcon className="rtl:[transform:var(--i18n-icon-flip-rtl)]" />
```

## Component contract

After this foundation is written, every component:

- Uses **logical properties** (`padding-inline-*`, `margin-block-*`)
  not physical (`padding-left`, `margin-top`)
- Wraps directional icons with the `rtl:` flip utility
- Reads from `--font-body` (which now includes script fallbacks)
  rather than declaring its own font stack
- Uses `text-align: start | end` not `left | right`

The Phase 3 lint sweep extends to flag physical-direction CSS
properties in component source.

## Audit hook

`/audit` gains an i18n plane (Plane 7, optional — only runs
when the project declares multilingual support):

- **Hardcoded direction strings**: any `padding-left:`,
  `margin-right:`, `text-align: left` in component code
- **Untranslated strings**: text in component code that doesn't
  go through the project's i18n function (`t("...")`,
  `<FormattedMessage>`, etc.) — heuristic, surfaces for review
- **Missing icon flip**: directional icons (`chevron-*`, `arrow-*`)
  rendered without the RTL-flip utility

## K-12 ed-tech notes

For multilingual K-12 contexts:
- **Family communications must support the family's preferred
  language**, set per-family in the platform.
- **Student-facing UI may differ from family-facing UI** — students
  often use English at school while families prefer their home
  language. The `dir` attribute is set per-route or per-region, not
  globally.
- **Numerals in transcripts and grades** typically stay in Western
  Arabic numerals (1, 2, 3) even in RTL contexts because that's what
  schools use officially. Use `unicode-bidi: isolate` on numeric
  spans to prevent reordering.
- **Required scripts at minimum**: latin, latin-ext (Spanish, Polish,
  Vietnamese), arabic (Arabic-speaking families), simplified-chinese
  (Mandarin-speaking families), and add others per district
  demographics.

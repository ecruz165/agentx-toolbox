Sync

Arguments: $ARGUMENTS
Merge edits from YAML task files back into tasks.json.

## Execution

```bash
npx taskmaster sync
npx taskmaster sync --dry-run
```

## Options

- `--dry-run`: preview changes without modifying tasks.json

## What It Does

1. Reads all YAML task files from the project's `tasks/` directory
2. Compares each field against the corresponding task in tasks.json
3. Applies YAML-side changes to tasks.json (YAML wins on conflict)
4. Reports which tasks and fields were updated

## When to Use

- After manually editing YAML task files
- After external tools modify YAML files
- To merge hand-tuned task descriptions back into the canonical store

## Important

- `tasks.json` is the single source of truth
- YAML files are projections generated by `npx taskmaster generate`
- The sync command is the ONLY way to flow YAML edits back to tasks.json
- Always review changes with `--dry-run` first
- After syncing, regenerate YAML files with `npx taskmaster generate` for consistency
